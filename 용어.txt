IDS(Intrusion Decetion System)
특정 패턴을 기반으로 공격자의 침입을 탐지한다.

IPS(Intrusion Prevent System)
침입방지시스템으로 인터넷 웜 등의 
악성코드 및 해킹에 기인한 유해트래픽 
차단 및 공격 탐지를 실시한다. 공격에 대한
웹 연결을 끊는 등 적극적인 솔루션이다.
OS나 애플리케이션의 취약점을 능동적으로
사전에 보완하고 웜이나 버퍼오버플로우,
특히 비정상적인 트래픽이나 알려지지 않은
공격까지 차단할 수 있다. 가장 큰 특징은
기업 외부에서 내부 네트워크로의 침입을 
방지하는 것이다.

*네트워크 기반 IPS(NIPS) 
방화벽처럼 네트워크에 인라인 모드로 설치
*호스트 기반 IPS
서버 애플리케이션을 담당하며 시큐어 OS
등과 비슷한 기능

웜 : 스스로를 복제하는 프로그램. 독자적으로 실행되며
종종 컴퓨터의 파일 전송 기능을 착취하도록 설계된다.
네트워크를 사용하여 자신의 복사본을 전송하며, 
네트워크를 손상시키고 대역폭을 잠식한다.

버퍼오버플로우 : 메모리를 다루는 데에 오류가 발생하여
잘못된 동작을 하는 프로그램 취약점이다. 프로세스가 
데이터를 버퍼에 저장할 때 프로그래머가 지정한 곳 바깥에 
저장하는 것을 의미한다. 벗어난 데이터는 인접 메모리를 
덮어쓰게 되며 이때 다른 데이터가 포함 될 수 있는데, 손상
받을 수 있는 데이터는 프로그램 변수와 프로그램 흐름 제어 데이터도
포함된다. 이로 인해 잘못된 프로그램 거동이 나타날 수 있으며, 
메모리 접근 오류, 잘못된 결과, 프로그램 종료, 또는 시스템 보안 
누설이 발생한다.

HTTP 
인터넷 상에서 정보를 주고 받기위한 프로토콜
클라이언트 서버 사이에 이루어지는 요청/응답 프로토콜
암호화되지 않는 방법으로 데이터 전송

HTTPS
보안이 강화된 HTTP
Hypertext Transfer Protocol Over Secure Socket Layer의 약자
모든 HTTP 요청과 응답 데이터는 네트워크로 보내지기 전에 암호화된다.
HTTPS는 HTTP의 하부에 SSL과 같은 보안 계층을 제공한다.

SSL(Secure Socket Layer)
보안 소켓 계층을 이르는 것으로, 인터넷 상에서 
데이터를 안정하게 전송하기 위한 인터넷 암호화
통신 프로토콜이다.

Snort
스노트는 자유 오픈 소스 네트워크 차단 시스템이자 네트워크
침입 탐지 시스템이다. 실시간 트래픽 분석과 IP에서의 패킷 로깅을
수행하는 능력을 갖는다. 프로토콜 분석, 내용 검색, 매칭을 수행

YARA 
Malware 조사 및 탐지에 주로 사용되는 도구. 텍스트 혹은 이진 패턴을 
기반으로 멀웨어 계열에 대한 설명을 작성하는 규칙 기반 접근 방식 제공

Malware 
컴퓨터를 손상시킬 목적으로 만들어진 소프트웨어를 말한다. 정상적인 
작동을 방해하거나 사용자의 디바이스를 감염시키도록 설계된 악성코드를
총 칭한다. ex) 웜, 트로이 목마, 스파이웨어, 키로거 등

In-Line
Sniper Engine을 통과한 패킷을 검사하고 실시간 차단이 되는 모드
In-Line이 아니라는 것은 차단하지 않는다는 의미

Span(Switched Port Analysis)
포트 미러링 혹은 포트 스패닝이라고 하며 스위치, 라우터 장비에서
수신되는 모든 프레임, 패킷들을 수신 포트를 제외한 모든 포트로 
의무적으로 브로드캐스팅하지 않기 때문에, 반드시 지정된 모니터링 
포트로 트래픽을 복사 할 수 있게 설정하는 기능.
탐지만 하고 화면에 차단이 나와도 차단은 않음

BYPASS 
광 스위치를 통해 Port0 - Port1 물리적으로 연결하여, network 직접
연결하는 효과를 가지게 됨. SNIPER 프로그램이 동작하지 않는 상태.
탐지도 안함. 제품에 제공되는 전원에 이상이 있을 경우, 제품에 탑재
되어 있는 O.S에 문제가 발생할 경우, 제품 엔진에 문제가 발생할 경우

 
FW(Firewall)
패킷의 IP와 PORT(TCP/UDP)를 기준으로
룰의 목적과 비교하여 차단한다.

APT(Advanced Persistent Threat)
기업 혹은 조직 등 특정 대상을 먼저 선정하고
내부의 취약한 시스템을 이용해 침투한 다음
오랜 시간에 걸쳐 다양한 공격 기법을 활요해
피해자도 모르게 공격하는 방식을 말한다. 크게
시도, 성공, 확산, 유출의 4단계로 나눌 수 있다.

RegEX & 정규표현식
특정한 규칙을 가진 문자열의 집합을 
표현하는데 사용하는 형식 언어이다.
문자열의 검색과 치환을 위해 지원.

HA(High Availability) & 고가용성
서버와 네트워크, 프로그램 등의 정보
시스템이 상당히 오랜 기간 동안 
지속적으로 정상 운영이 가능한 성질.

Proxy
프록시는 클라이언트와 서버 사이에 
중계기 역활을 하며, 서버가 외부에 
직접적으로 노출되지 않도록 해줍니다.

Transparent Proxy(forward Proxy)
외부 클라이언트가 서버IP를 보고 
접속하는 방식. 서버IP로 접속하기 때문에
DNS설정이 필요없으며, CSLB(Cache
Server Load Balancing)을 지원하는
L4 스위치와 연결되어야 한다.

CSLB(Cache Server Load Balancing)
여러 서버에 걸쳐 웹 사이트에 트래픽을
분산시키고 내장 캐시에서 신속하게
컨텐츠를 제공하며 문제가 발생할 때
서버가 자동으로 순환되지 않도록한다.
플랫폼을 수평으로 확장하고 속도를 높이고
연속성을 보장 할 수 있다.

Zero-Day Attack & 제로 데이 공격
컴퓨터 소프트웨어 취약점을 공격하는
기술적 위협으로, 해당 취약점에 대한
패치가 나오지 않을 시점에서 이루어지는
공격을 말한다.

C&C Server(Command&Control)
감염된 좀비 PC가 해커가 원하는 공격을
수행하도록 원격지에서 명령을 내리거나,
악성코드를 제어하는 서버를 말한다.

ERP Server(Enterprise Resource Planning)
구매와 생산과리, 물류, 판매, 회계 등의
기업 활동 전바에 걸친 업무를 통합하여
경영자원을 최적화한다는 개념으로 
기업의 모든 자원에 대해서 계획, 활용
하는 것이라고 볼 수 있으며 '전사적
자원 관리'라고도 한다.

DRM Server(Digital Rights Management)
음악 파일이나 동영상 파일, 문서 파일에
대한 권리를 제공하는 기술로 허가된 
사용자 이외에는 접근할 수 없도록 제어하는
기술이다.

DMZ(Demilitarized Zone)
조직의 내부 네트워크와 외부 네트워크
사이에 위치한 서브넷이다. 내부 네트워크와
외부 네트쿼으가 DMZ로 연결할 수 있도록
허용하면서도, DMZ 내의 컴퓨터는
오직 외부 네트워크에만 연결할 수 있도록 한다.
DMZ 안에 있는 호스트들은 내부 네트워크로 
연결할 수 없다. 메일서버, 웹서버, DNS 서버와 
같이 외부에서 접근되어야 할 필요가 있는
서버들을 위해 사용된다.

Dual-Stack
IPv4/IPv6 듀얼스택은 IPv6 노드가 
IPv4 전용 노드와 호환성을 유지하는 
가장 쉬운 방법이다.

BPS(bits per second)
통신 속도의 단위로 1초간에 송수신할 수 있는
비트수를 나타낸다. 해당 회선/포트 내에서
보내거나 받을 수 있는 총 비트의 수.

PPS(packets per second)
근거리 통신망의 전송 속도를 나타내는 단위
초당 몇 개의 패킷을 처리할 수 있는지

QoS(Quality of Service)
다른 응용 프로그램, 사용자, 데이터 흐름 등에
우선 순위를 정하여, 데이터 전송에 특정 수준의
성능을 보장하기 위한 능력을 말한다.
네트워크에서 QoS라 함은 
1. 중요한 패킷과 그렇지 않은 패킷을 구분하고
2. 망에 혼잡이 발생하였을 때 중요한 패킷을 먼저 보내주고,
    그렇지 않은 패킷은 drop하는 행위입니다. 

5-tuple 
포함되는 필드는 출발지 IP, 목적지 IP, 프로토콜 ID, 출발지 포트, 목적지 포트이다.

IP Pool
인터넷 주소 구조의 맥락에서, 주소 풀은
IP 주소 할당 계층의 모든 수준에서
사용 가능한 인터넷 프로토콜 주소 집합이다.

Slowloris 공격
HTTP Header 정보를 비정상적으로 조작하여
웹서버가 온전한 Header 정보가 올 때 까지
기다리도록 한다. 서버가 연결 상태를 유지할 수 있는
가용자원은 한계가 있으므로 임계치를 넘어가면 다른
정상적인 접근을 거부하게 된다.

POST 공격
L3기반의 DDoS 공격으로, HTTP 헤더값을
일부분만 보내서 웹서버와의 커넥션을 장시간
유지하여 정상적인 접속을 차단하는 공격이다.
Slowloris와 유사하나 Apche 뿐만 아닌 IIS 등에
모두 영향을 미치며, 헤더의 Content-Length 필드에
임의의 큰 값을 설정하여 전송한 후, 소량의 데이터를
느린 속도로 전송하여 연결을 유지하면서
공격한다는 특징을 가지고 있다.

VoIP & 음성 인터넷 프로토콜
인터넷과 같은 인터넷 프로토콜 네트워크를 통해
음성 통신과 멀티미디어 세션의 전달을 위한 기술들
의 모임을 가리키는 용어.

Rate Limit
대표적인 혼잡 제어 기법으로 traffic ratef를
제한합니다. 네트워크 장비의 입력 포트에서 
구현되며, DDoS 공격, 무차별 암호 
대입 시도 및 애플리케이션 레이어를 
대상으로 하는 기타 악의적인 동작
으로부터 지켜줍니다.

ESM(Enterprise Security management)
기업보안관리로 방화벽, 침입 탐지 시스템, 가상 사설망 등의 보안 솔루션을
하나로 모은 통합 보안 관리 시스템으로 기업이 개별적으로 도입해 온 각종 
보안 솔루션을 서버단에서 하나의 콘솔로 관리할 수 있게 한다.

TMS(Threat Management System)
위협관리 시스템으로 IDS/IPS 등에서 
수집된 탐지/차단 로그의 발생량과 
룰(로그)별로 산정된 위험도의 조합에 의해
위협수준을 결정해 주는 솔루션이다.

RMS(Risk Management System)
위험관리 시스템으로 보호대상 정보자산의
구성 현황을 파악하고 자산별 운영체제,
애플리케이션, 가동 서비스 및 이하 관련된
취약점 현황을 파악 및 분석한 후, 이 정보를
DB화하여 위험 발생 요인의 체계적
관리를 목적으로 하는 시스템이다.

Cyber Kill Chain
사이버 공격을 프로세스 상으로 분석해 각 공격
단계에서 조직에게 가해지는 위협 요소들을 
파악하고 공격자의 목적과 의도, 활동을 분쇄,
완화시켜 조직의 회복 탄력성을 확보하는 전략입니다.
한 마디로 공격자의 입장에서 사이버 공격 활동을
파악, 분석해 공격 단계 별로 조직에게 가해지는
위협 요소를 제거하거나 완화하자는 것입니다.

SIEM(Security Informaiton & Event Management)
빅데이터의 방대한 정보 속에서 단순한 로그 수집 및
분석이 아닌 사후에 추적 등이 가능하도록 상관분석과 
포렌식 기능을 제공해주는 지능적 위협에 대한 조기
경고 모니터링 체계. 기업 내 정보시스템의 보안 정보
및 이벤트 관리를 통해 내부 및 외부 위협을 모니터링 
함으로써 외부의 공격은 물론, 내부 정보유출 또한
방지하는 것

SNMP(Simple Network Management Protocol)
간이 망 관리 프로토콜으로 IP 네트워크상의 장치
로부터 정보를 수집 및 관리하며, 또한 정보를 수정
하여 장치의 동작을 변경하는 데에 사용되는 인터넷
표준 프로토콜이다. SNMP를 지원하는 대표적인 장치에는
라우터, 스위치, 서버, 워크스테이션, 프린터, 모뎀 랙 등이 
포함된다. 네트워크 모니터링의 목적으로 네트워크 관리에서
널리 사용된다.

데몬(Daemon)
리눅스 시스템이 처음 가동될 때 실행되는 백그라운드 프로세스의 일종이며
사용자의 요청을 기다리고 있다가 요청이 발생하면 이에 적절히 대응하는 
리스너와 같은 역할을 한다. 즉, 메모리에 상주하면서 특정 요청이 오면 즉시
대응 할 수 있도록 대기중인 프로세스를 말한다. MS윈도우의 서비스와 유사하다.

리눅스에서 데몬을 실행하는 방법은 크게 두가지가 있다.
Standlone type daemon
-독립적으로 수행되며 서비스 요청에 응답하기 위해 항시 메모리에 상주하는 데몬이다.

Super daemon & inetd type deamon
-inetd라고 하는 특별한 데몬에 의해 간접적으로 실행되는 데몬이다.
보안상의 이유로 xinetd로 수정되었으며 슈퍼데몬이라고도 한다.
inetd에 어떠한 요청이 들어오면 inetd는 해당되는 데몬을 메모리에 
올리고 요청을 처리하게 된다.

WAF(Web Application Firewall)
웹방화벽으로 일반적인 네트워크 방화벽과는 달리 웹애플리케이션
보안에 특화되어 개발된 솔루션입니다. SQL Injection, Cross-Site Scripting
(XSS)등과 같은 웹 공격을 탐지하고 차단한느 것입니다.

SQL Injection
SQL 삽입은 응용 프로그램 보안 상의 
허점을 의도적으로 이용해, 악의적인 SQL문을
실행되게 함으로써 데이터베이스를 비정상적으로
조작하는 코드 인젝션 공격 방법이다.

코드 인젝션
유효하지 않은 데이터를 실행함으로써 야기되는 소프트웨어
버그의 부당한 사용이다. 삽입은 공격자에 의해서 취약한 컴퓨터
프로그램 코드를 삽입하고 실행을 변경하는 방식으로 이용된다.


스푸핑(Spoofing)
해커가 악용하고자하는 호스트의 IP 어드레스를 바꾸어서 이를 통해 
해킹을 하는 것이 IP 스푸핑이다. 네트워크 시스템에서 서로 신뢰관계
에 있는  A,B 두 시스템간에는 A 시스템의 어카운트를 가지고 B 시스템을
엑세스 할 수있다. 이는 네트워크에서 신뢰관계를 형성하는 서비스가 
네트워크 주소에 기반하여 이를 인증하기 때문이다. IP 스푸핑은 이 신뢰
관계에 있는 두 시스템사이에서 해커의 호스트를 마치 하나의 신뢰관계
에 있는 호스트인 것처럼 속이는 것이다. 인터넷 프로토콜인 TCP/IP의 
구조적 결함을 이용해 사용자의 시스템 권한을 획득한 뒤, 정보를 빼가는
 해킹 수법을 말한다.

ACL(Access Control List)란 interface로 들어오고 나가는 패킷을 분석해서
정해진 규칙에 따라 패킷을 전송 또는 차단함으로써 네트워크 접근 제어 
라우터의 방화벽이다.

Kernel Shared Memory
프로세스간에 데이터를 공유하고자 할때 사용하는 메모리로 커널에 의해서 관리
되어집니다. 공유메모리는 최초로 공유메모리를 만드는 프로세스에 의해서 만들어
집니다. 이렇게 만들어진 메모리는 커널이 관리해 줍니다. 한번 만들어진 공유 메모리 
공간은 직접 삭제를 하거나 리눅스 시스템이 재부팅을 하거나 해야지만 없어집니다.
모든 프로세스가 더 이상 공유 메모리를 사용하지 않는다고 자동 삭제되는 일은 결코 
없습니다.

NAT(Network address translation)
네트워크 주소 변환은 컴퓨터 네트워킹에서 쓰이는 용어로서, IP 패킷의 TCP/UDP 
포트의 숫자와 소스 및 목적지의 IP 주소 등을 재기록하면서 라우터를 통해 네트워크
트래픽을 주고받는 기술을 말한다. 패킷에 변화가 생기기 때문에 IP나 TCP/UDP의 
체크섬도 다시 재기록해야 한다. NAT를 이용하는 이유는 대게 사설 네트워크에 속한
여러 개의 호스트가 하나의 공인 IP 주소를 사용하여 인터넷에 접속하기 위함이다.

SMB(Server Message Block)
서버 메시지 블록은 도스나 윈도우에서 파일이나 디렉터리 및 주변 장치들을 
공유하는데 사용되는 메시지 형식이다.

PCI-E
입출력을 위한 직렬 구조의 인터페이스이며 인텔 주도하에 만들어졌다.

TAP(Test Acess Port)
네트워크 상에서 전송되는 데이터를 full duplex mode에서 패킷의 흐름에
전혀 영향을 주지 않고, 패킷 손실없이 패킷을 모니터링할 수 있는 장비입니다.
In-Line 한 구간에 이동하는 패킷 데이터를 복사하여 모니터 장비로 보내 주는
역할을 하는 가장 기본적인 장비
Optical Spilter를 이용해여 트래픽을 미러링 할 수 있는 장비

Watchdog Time
Sniper IPS/DDX 데몬에서 Inline을 유지하기 위한 Signal 유지 시간
데몬이 다운될 경우 WDT Time만큼 Inline을 유지 

NIC RISER CARD
메인 보드에 기능을 추가하기 위해 수직으로 장착하는 소형의 반도체 기판
어댑터 카드를 위한 추가 슬롯을 제공하는 보드이다.

LB(Load Balancing)
하나의 인터넷 서비스가 발생하는 트래픽이 많을 때 여러 대의 서버가
분산처리하여 서버의 로드율 증가, 부하량, 속도저하 등 을 고려하여
적절히 분산처리하여 해결해주는 서비스입니다.

L2 -> Mac주소를 바탕으로 LB
L3 -> IP주소를 바탕으로 LB
L4 -> Transport Layer(IP와 Port) Level에서 LB, TCP,UDP
L7 -> Application Layer(사용자의 Request) Level에서 LB
        HTTP, HTTPS, FTP

RR(Round Robin Scheduling)
시분할 시스템을 위해 설계된 선점형 스케줄링의 하나로서, 프로세스들
사이에 우선순위를 두지 않고, 순서대로 시간단위로 CPU를 할당하는 
방식의  CPU 스케줄링 알고리즘이다.

SSH(Secure Shell)
시큐어 셀이란 네트워크 상의 다른 컴퓨터에 로그인하거나
원격 시스템에서 명령을 실행하고 다른 시스템으로 파일을
복사할 수 있도록 해 주는 응용 프로그램 또는 그 프로토콜
을 가리킨다.

레드마인(Redmine)
오픈 소스 프로그램으로 웹 기반의 프로젝트 관리와 버그 추적 기능을 제공하는 도구이다.
화면기반의 프로젝트 관리에 도움이 되도록 달력과 간트 차트를 제공하고 일정관리 기능을
제공한다. 또한 레드만인은 통합된 프로젝트관리 기능과 이슈추적, 여러가지 형상 관리 기능을 
제공한다. 루비 온 레이즐에 기반하여 작성되었고 멀티 플랫폼을 지원하며 여러가지 종류의 
데이터베이스 및 34개의 언어를 지원한다.

루비 온 레일즈(Ruby on Rails)
루비로 작성된 MVC 패턴을 이용하는 오픈 소스 웹 프레임워크이다. 데이터베이스를 이용한 웹 
애플리케이션을 개발할 때 반복되는 코드를 대폭 줄여 개발 기간으 단축하는 것으로 인기를 끌고
있다. 모델, 뷰, 컨트롤러의 템플릿 생성 및 테스트 등 자동화하는 툴, 테스트를 위한 웹서버 등을 
포함하고 있다. 루비 온 레일즈가 기반을 두고 있는 ActiveRecord는 데이터베이스 작업을 추상화하여
생성/읽기/갱신/삭제 작업을 자동화 및 단순화 시켜준다.

루비(Ruby)
마츠모토 유키히로가 개발한 동적 객체 지향 스크립트 프로그래밍 언어이다. 루비는 순수 객체 지향 언어라,
정수나 문자열 등을 포함한 데이터 형식 등 모든 것이 객체이다. 기능에는 클래스 정의, 가비지 컬렉션, 강력한
정규 표현식 처리, 다중 스레드, 예회 처리, 반복, 클로저, Mixin, 연산자 오버로드 등이 있다.

SQLite 
MySQL이나 PostgreSQL와 같은 데이터베이스 관리 시스템이지만, 서버가 아니라 응용 프로그램에 넣어 사용하는 
비교적 가벼운 데이터베이스이다. 일반적인 RDBMS에 비해 대규모 작업에는 적합하지 않지만, 중소 규모라면 속도에 
손색이 없다. 또 API는 단순히 라이브러리를 호출하는 것만 있으며, 데이터를 저장하는데 하나의 파일만을 사용하는 
것이 특징이다. 구글 안드로이드 운영 체제에 기본 탑재된 데이터베이스기이도 하다.

세그먼트(Segment)
네트워크 1구간을 구성하는 단위
NIC Port 0(Inbound), 1(Outbound)을 쌍으로 단위

세션(Session)
반영구적이고 상호작용적인 정보 교환을 젠제하는 둘 이상의 통신 장치나
컴퓨터와 사용자 간의 대화나 송수신 연결상태를 의미하는 보안적인 다이얼로그 
및 시간대를 가리킨다. 따라서 세션은 연결 상태를 유지하는것보다 연결상태의 
안정성을 더 중요시 하게된다.

스캐닝
공격자가 공격을 수행하기에 앞서 타겟 서버에 대한 정보 및 유저의 정보 등을
획득하기 위한 단계이다. 스캐닝을 통해 얻은 정보를 이용하여 공격자는 수행할
공격을 특징짓는다.

패시브 스캐닝(Passive Scanning)
패시브 스캐닝은 공격을 수행할 대상에 대해 제 3사이트(구글, whois, 취업관련 사이트 등)을
이용하여 정보를 획득하는 스캐닝 기법을 총칭한다.

액티브 스캐닝(Active Scanning)
공격을 수행할 대상에 직접 프로그램 혹은 스크립트 등을 이용하여 대상에 대한 정보
(서버 정보, ip 관련 정보, 취약점 정보 등)을 획득하는 스캐닝 기법이다.

디렉토리 서비스
컴퓨터 네트워크의 사용자와 네트워크 자원에 대한 정보를 저장하고 조직하는
응용 소프트웨어이다. 네트워크 관리자가 여러 사용자들이 자원에 접근할
수 있게 도와준다. 사용자와 공유된 자원 사이의 추상 계층으로 동작한다.
 
LDAP(Lightweight Directory Access Protocol)
조직이나, 개체, 그리고 인터넷이나 기업 내의 인트라넷 등 네트워크 상에
있는 파일이나 장치들과 같은 자원 등의 위치를 찾을 수 있게 해주는 
소프트웨어 프로토콜이다. TCP/IP 위에서 디렉터리 서비스를 조회하고 
수정하는 응용 프로토콜이다.

액티브 디렉터리(Active Directory)
마이크로소프트가 윈도우용 환경에서 사용하기 위해 개발한 LDAP 디렉터리 
서비스의 기능이다. 주 목적은 윈도우 기반 의 컴퓨터들을 위한 인증 서비스를
제공하는 것이다. 주로 윈도우 환경에서 동일한 데이터베이스를 사용한다.

CGI(Common Gateway Interface)
CGI는 HTTP를 해석하는 Informaiton 서버와 클라이언트 프로그램 사이의 통신을 가능하게 해주는
일종의 인터페이스이다. TCP/IP는 서버와 CGI 스크립트에 의해 사요오디는 통신 프로토콜이고,
기본 포트는 80번 이지만, 다른 포트가 사용될 수도 있다.
클라이언트에서 서버로, 서버에서 클라이언트로 이동되는 데이터의 통로로 작동된다.

RTS/CTS - 하드웨어 제어 신호에 의한 흐름 제어 방식
RTS는 자신의 수신버퍼 상태를 알려주는 신호이고
CTS는 상대의 수신버퍼의 상태를 받는 신호이다.

XON/XOFF - 소프트웨어 적으로 송수신 데이터에 대한 흐름제어이다.
컴퓨텅과 비동기 직렬 접속되어 있는 다른 장치들 간에 데이터 흐름을
제어하기 위한 프로토콜이다. 예를 들어, 컴퓨터는 보통 프린터가 찍을 
수 있는 것보다 빠르게 데이터를 전송한다. 프린터는 프린터가 컴퓨터의
속도를 따라잡을 수 있을 때까지 데이터를 임시 저장하기 위한 버퍼를 
가지고 있다. 만약 프린터가 컴퓨터의 전송속도를 따라잡기 전에 버퍼가
꽉 차게되면, 프린터 내의 작은 마이크로프로세서가 데이터 전송을 중지라
하라는 신호인 Xoff 신호를 컴퓨터에 보낸다. 그 후 데이터가 충분히 인쇄되고
버퍼의 저장공간에 여유가 생기면, 프린터는 컴퓨터에게 데이터 전송을 다시 
재개하라는 의미로 Xon 신호를 보낸다. X는 transmitter의 약자로 Xon 또는
Xoff는 신호는 송신장치를 켜거나 끄기 위한 것이다.

DTR/DSR
DTR은 해당 장비가 통신준비가 되었다는 것을 상대방의 장비에게 알려주는 신호이고,
DSRR은 상대장비가 통신준비가 되어있는 지를 감시하는 신호선입니다.
보통은 이 신호선들을 이용해서 상대장비와 연결되어있는지를 체크합니다.

Baud Rate
1초당의 변조횟수 혹은 초당 얼마나 많은 심볼(의미 있는 데이터 묶음)을 
전송할 수 있는 가를 나타내는 말이다.
이론적인 통신 단위로 초당 신호 요소의 수를 나타낸다.
시리얼 통신에서는 Dada bit가 8-bit를 사용하므로 이를 하나의 심볼이라고 이야기할 수 있다.
1개의 심볼(또는 character)는 8개의 bit 정보와 같다.

SWAP
스왑 파티션은 시스템 메모리가 부족할 경우 하드 디스크의 일부 공간을 활용하여
작업을 계속 진행할 수 있도록 돕는 영역입니다. 스왑 영역을 사용하게되면 메모리에
처리되어야 할 작업이 하드 디스크에서 처리되기 때문에 프로그램 실행이나 명령어 
처리에 있어서 인터럽트가 발생하지 않는다는 이점이 있습니다. 하지만 하드 디스크
영역을 사용하기 때문에 RAM의 빠른 처리 속도에 미치지 못하는 수준에서 작업이 진행

PPS(Packet Per Second)
초당 몇 packet을 전송하는지를 나타낸다.

CPS(Connections Per Second)
초당 TCP 커넥션 처리 능력을 나타낸다.
TCP Connection + HTTP Transaction + TCP Closed

TPS(Transaction Per Second)
초당 HTTP 트랜잭션 처리 능력을 나타낸다.
HTTP Transaction

소프트웨어 구성 관리 또는 형상 관리
소프트웨어의 변경사항을 체계적으로 추적하고 통제하는 것으로,
형상 관리는 일반적인 단순 버전관리 기반의 소프트웨어 운용을 
좀 더 포괄적인 학술 분야의 형태로 넓히는 근간을 이야기한다.
일반적으러 형상 항목이라는 형태로 작업 산출물을 선정하고, 
형상 항목 간의 변경 사항 추적과 통제 정책을 수립하고 관리한다.

FLAG
다른 프로그램에게 약속된 신호를 남기기 위한 용도로 프로그램에서
사용되는 미리 정의된 비트를 의미
TCP 헤더에는 1비트짜리 플레그 6개가 위치해 있다.

SYN(Synchronization) 연결 요청 플레그
통신 시작 시 세션을 연결하기 위한 플레그

ACK(Acknowledgement) 응답 플레그
송신측으로부터 패킷을 잘 받았다는 걸 알려주기 위한 플레그

FIN(Finish) 연결 종료 플레그
더 이상 전송할 데이터가 없고 세션 연결을 종료시키겠다는 플레그

RST(Reset) 연결 재설정 플레그
비정상적인 세션을 끊기위해 연결을 재설정하는 과정

PSH(Push) 넣기 플레그
버퍼가 채워지기를 기다리지 않고 받는 즉시 전달한다.
버퍼링 없이 7Layer Applictaion Layer의 응용프로그램에게
바로 전달되는 플레그

URG(Urgent) 긴급 데이터 플레그
긴급한 데이터의 우선순위를 다른 데이터의 우선순위를
높여 긴급하게 전달하는 플레그
